# 要件定義書（第一段階：要件定義・環境構築フェーズ）

## 1. プロジェクト概要
小説や物語制作を支援する **AIシナリオエディター** のプロトタイプを構築する。  
執筆者が自然言語で物語を進める中で、AIが補完・提案を行う執筆補助ツールをWeb上で利用可能にすることを目的とする。

本フェーズでは、今後のMVP開発に向けた **技術選定・環境整備・要件の具体化** を行い、スムーズな開発着手を目指す。

---

## 2. 第一段階（スコープ）
以下の項目を第一マイルストーンの対象範囲とする。

### (1) 要件定義
- クライアント提供ドキュメント（HackMD）に基づき、実現優先度・MVP範囲を具体化
- 主要機能（AI補完 / エディター / ログイン / API管理など）の仕様確定
- データ構造・API設計方針の初期案策定
- MVPとして達成すべきKPI（例：物語生成完結率、レスポンス速度など）を設定

### (2) 環境構築
- **リポジトリ管理**：GitHubリポジトリ（pxrllc/ArcanaEditor）で管理  
- **フロントエンド環境構築**：Next.js + TypeScript  
- **バックエンド構築**：Supabaseをベースに認証・DB連携を実装可能な状態に整備  
- **デプロイ環境**：Vercel（ステージング）を設定  
- **AI接続基盤準備**：ChatGPTまたはClaude APIを統合できる構成を設計  
  - APIキーをクライアント側で発行し、制限管理の方針を策定  
- **コラボレーション環境**
  - Issueベースのタスク管理（GitHub）
  - FigmaによるUI設計共有（クライアント側利用ツールに準拠）

---

## 3. 想定MVP範囲（第二段階で開発予定）
- AI補完付きエディター機能  
- シンプルなログイン / ホワイトリスト制御  
- ユーザー単位のAPI使用回数制限  
- 物語作成・保存・再編集機能  
- プロンプト履歴管理・AI応答制御  

※上記は要件定義フェーズにて優先順位を再確認し、実装範囲を確定する。

---

## 4. 使用予定技術スタック

| 区分 | 技術 | 用途 |
|------|------|------|
| フロントエンド | Next.js / React / TypeScript | UI構築・SPA |
| スタイリング | Tailwind CSS | スタイル統一・レスポンシブ対応 |
| バックエンド | Supabase | DB・認証・API連携 |
| AI連携 | OpenAI GPT-4 / Claude API | シナリオ補完・プロンプト応答 |
| デプロイ | Vercel | CI/CD・ステージング運用 |
| デザイン | Figma | UI設計・画面共有 |
| 管理 | GitHub | リポジトリ・Issue管理 |
